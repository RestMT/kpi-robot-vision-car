Навчальний проєкт для керування роботом-автомобілем з камерою ESP32-CAM через Wi-Fi.

Головна ідея — продемонструвати способи підключення до робота, отримання відеопотоку з камери та надсилання команд руху з Python-програми або власного графічного інтерфейсу.

У репозиторії є дві окремі програми:

- `main.py` та її скомпільована версія `KPI Robot Car.exe` - основна тестова програма для перевірки працездатності робота, правильності підключення, пошуку робота в мережі, роботи відеопотоку та потоку команд.
- `simple_robot_demo.py` - спрощена навчальна програма з графічним інтерфейсом на Tkinter. Вона показує мінімальний приклад того, як можна написати власний простий застосунок для керування роботом.

Основні адреси (ендпоінти) API:

```text
WebSocket-команди: ws://<ESP32_IP>/ws
Відеопотік:        http://<ESP32_IP>:81/stream
```

> [!TIP]
> Якщо потрібно швидко перевірити робота - запускайте `main.py`.
> Якщо потрібно подивитися найпростіший приклад власного GUI - відкривайте `simple_robot_demo.py`.

## Що вміє робот

Робот підтримує:

- рух вперед, назад, ліворуч і праворуч;
- зупинку;
- зміну швидкості руху;
- увімкнення та вимкнення світлодіода (фари);
- перевірку зв’язку;
- отримання діагностичного статусу;
- передавання відеопотоку з камери;
- автоматичний пошук робота в локальній Wi-Fi мережі.

## Швидкий старт

### 1. Встановіть залежності

Рекомендовано використовувати Python 3.12.

```bash
pip install -r requirements.txt
```

### 2. Запустіть основну тестову програму

```bash
python main.py
```

У вікні програми:

1. натиснути `Знайти робота`;
2. вибрати потрібного робота зі списку, якщо їх кілька;
3. натиснути `Підключитися`;
4. керувати рухом робота за допомогою кнопок;
5. одночасно переглядати відео з камери.

### 3. Або запустіть мінімальний навчальний приклад

```bash
python simple_robot_demo.py
```

Файл `simple_robot_demo.py` - це простий приклад. Він показує базову логіку підключення до робота, надсилання команд і виведення відео у вікні Tkinter.

## Основні адреси

У прикладах нижче замість `<ESP32_IP>` потрібно підставити IP-адресу вашого робота.

Наприклад, якщо ІР-адреса робота:

```text
192.168.0.1
```

Тоді адреси потоків команд та відео будуть такими:

```text
ws://192.168.0.1/ws
http://192.168.0.1:81/stream
```

| Призначення                | Адреса |
|----------------------------|---|
| Команди для робота         | `ws://<ESP32_IP>/ws` |
| Відеопотік                 | `http://<ESP32_IP>:81/stream` |
| Діагностика через браузер  | `http://<ESP32_IP>/status` |
| Довідка API через браузер  | `http://<ESP32_IP>/api` |
| Скидання Wi-Fi налаштувань | `http://<ESP32_IP>/wifi/reset?confirm=1` |

## WebSocket API команд

> [!IMPORTANT]
> Усі команди надсилаються як звичайний текст.
> Для команди руху достатньо одного слова: `forward`, `backward`, `left`, `right` або `stop`.

Робот приймає прості текстові команди через WebSocket:

```text
ws://<ESP32_IP>/ws
```

### Команди руху

```text
forward
backward
left
right
stop
```

| Команда | Що робить |
|---|---|
| `forward` | рух вперед |
| `backward` | рух назад |
| `left` | поворот ліворуч |
| `right` | поворот праворуч |
| `stop` | зупинка |

Робот повинен рухатися лише тоді, коли програма регулярно надсилає команди руху. Інкаше через 0,5 секунд після надходження останньої команди робот автоматично зупиниться. Після відпускання кнопки обов’язково потрібно надіслати:

```text
stop
```

### Встановлення швидкості

```text
speed:<value>
```

де `<value>` — число від `85` до `255`. Чим вище число, тим вища швидкість.

Приклад:

```text
speed:250
```

Рекомендоване початкове значення:

```text
speed:170
```

### Світлодіод (фара)

```text
led:on
led:off
```

| Команда | Що робить |
|---|---|
| `led:on` | увімкнути світлодіод |
| `led:off` | вимкнути світлодіод |

### Команди для діагностики

```text
ping
status
```

| Команда | Очікувана відповідь |
|---|---|
| `ping` | `PONG` |
| `status` | JSON зі станом робота |

Приклад відповіді на `status`:

```json
{
  "ok": true,
  "robot_id": "A1B2C3D4E5F6",
  "robot_name": "Robot-E5F6",
  "speed": 170,
  "moving": false,
  "wifi_connected": true,
  "rssi": -55,
  "fps": 24.8
}
```

## Приклад підключення через WebSocket

У цьому прикладі показано варіант підключення до робота, задання швидкості, надсилання команду руху та зупинки робота.

```python
from time import sleep
from websocket import create_connection

ROBOT_IP = "192.168.0.1"

robot = create_connection(f"ws://{ROBOT_IP}/ws", timeout=2)

robot.send("ping")
print(robot.recv())  # PONG

robot.send("speed:170")
print(robot.recv())  # OK

robot.send("forward")
print(robot.recv())  # OK

sleep(1)

robot.send("stop")
print(robot.recv())  # OK

robot.close()
```

## Приклад керування кнопкою

Для кнопок руху важливо розділити дві дії:

```text
кнопку натиснуто  -> forward / backward / left / right
кнопку відпущено -> stop
```

Наприклад, функція для надсилання команди руху вперед та функція для зупинки можуть виглядати так:

```python
def start_forward():
    robot.send("forward")
    print(robot.recv())


def stop_robot():
    robot.send("stop")
    print(robot.recv())
```

У графічному інтерфейсі це означає:

```text
натиснули кнопку "Вперед"  -> виклик start_forward()
відпустили кнопку "Вперед" -> виклик stop_robot()
```

## Отримання відеопотоку через OpenCV

Відео з камери робота доступне за адресою:

```text
http://<ESP32_IP>:81/stream
```

Мінімальний приклад отримання відео через OpenCV та його виведення:

```python
import cv2

ROBOT_IP = "192.168.0.1"
VIDEO_URL = f"http://{ROBOT_IP}:81/stream"

video = cv2.VideoCapture(VIDEO_URL)

while True:
    ok, frame = video.read()

    if not ok:
        print("Не вдалося отримати кадр")
        break

    cv2.imshow("Robot camera", frame)

    if cv2.waitKey(1) == 27:  # Esc
        break

video.release()
cv2.destroyAllWindows()
```

В цьому прикладі відкривається окреме вікно OpenCV, в якому показується відео з камери робота.

---

## Мінімальний приклад: відео та команди

Нижче наведено дуже коротку програму, яка одночасно читає відеопотік та надсилає команди роботу.

```python
import cv2
from time import sleep
from websocket import create_connection

ROBOT_IP = "192.168.0.1"

robot = create_connection(f"ws://{ROBOT_IP}/ws", timeout=2)
video = cv2.VideoCapture(f"http://{ROBOT_IP}:81/stream")

robot.send("speed:170")
print(robot.recv())

robot.send("forward")
print(robot.recv())

for _ in range(50):
    ok, frame = video.read()

    if ok:
        cv2.imshow("Robot camera", frame)

    if cv2.waitKey(1) == 27:
        break

robot.send("stop")
print(robot.recv())

video.release()
robot.close()
cv2.destroyAllWindows()
```

Це лише навчальний приклад. У реальній програмі команду `stop` потрібно надсилати одразу після відпускання кнопки руху.

## Автоматичний пошук робота в мережі

Робот може оголошувати себе в локальній мережі через UDP broadcast.

Формат повідомлення:

```text
KPI_ROBOT_CAR;id=<robot_id>;ip=<ip>;ws=80;stream=81;name=<robot_name>
```

Приклад:

```text
KPI_ROBOT_CAR;id=A1B2C3D4E5F6;ip=192.168.0.1;ws=80;stream=81;name=Robot-E5F6
```

У основній програмі `main.py` натисніть кнопку:

```text
Знайти робота
```

Якщо комп’ютер і робот перебувають в одній Wi-Fi мережі, програма покаже знайдених роботів у списку. Якщо роботів кілька, виберіть потрібного перед підключенням у програмі для тестування.

Приклад списку:

```text
Robot-E5F6 — 192.168.0.1 — ID A1B2C3D4E5F6
Robot-91A0 — 192.168.0.1 — ID B7C8D9E091A0
```

## Скидання Wi-Fi налаштувань

Якщо потрібно підключити робота до іншої Wi-Fi мережі, можна скинути збережені Wi-Fi налаштування.

У графічній програмі `main.py` або `KPI Robot Car.exe` натисніть:

```text
Скинути Wi-Fi
```

Або перейдіть у браузері за таким посиланням:

```text
http://<ESP32_IP>/wifi/reset?confirm=1
```

Після скидання робот створить власну Wi-Fi точку доступу та запустить режим початкового налаштування Wi-Fi. Далі підключіться до його точки доступу та передайте нові SSID і пароль через графічний інтерфейс `main.py` або `KPI Robot Car.exe`.

Запуск програми:

### Варіант 1. Перевірка і налаштування робота через `main.py`

Використовуйте `main.py` або `KPI Robot Car.exe`, коли потрібно перевірити, чи робот правильно підключений, чи працює відеопотік, WebSocket-команди, пошук у мережі та службові функції.

1. Увімкніть робота.
2. Переконайтеся, що комп’ютер і робот перебувають в одній Wi-Fi мережі.
3. Запустіть основну тестову програму:

   ```bash
   python main.py
   ```
   або відкрийте `KPI Robot Car.exe`.
4. Натисніть `Знайти робота`.
5. Виберіть потрібного робота, якщо у списку їх кілька.
6. Натисніть `Підключитися`.
7. За потреби, змініть швидкість руху робота.
8. Натискайте та утримуйте кнопки руху.
9. Відпускайте кнопку руху для зупинки.
10. Одночасно переглядайте відео з камери.
11. За потреби перевірте світлодіод (фару), команди `ping`, `status`, пошук робота та скидання Wi-Fi налаштувань.

### Варіант 2. Мінімальний навчальний приклад через `simple_robot_demo.py`

Використовуйте `simple_robot_demo.py`, коли потрібно побачити максимально простий приклад власної програми для керування роботом. Цей файл не повторює всі можливості `main.py` (`KPI Robot Car.exe`), а показує базові принципи: підключення, роботу з відео і командами руху.

1. Увімкніть робота.
2. Переконайтеся, що комп’ютер і робот перебувають в одній Wi-Fi мережі.
3. Дізнайтеся IP-адресу робота через `main.py` (`KPI Robot Car.exe`), сторінку маршрутизатора або інший доступний спосіб.
4. Запустіть навчальний приклад:

   ```bash
   python simple_robot_demo.py
   ```

5. Введіть IP-адресу робота у відповідне поле.
6. Натисніть `Підключитися`.
7. За потреби, натисніть `Встановити швидкість`, щоб надіслати команду `speed:<value>`.
8. Натисніть `Запустити відео`, щоб отримати відеопотік з камери.
9. Натискайте та утримуйте кнопки руху.
10. Відпускайте кнопку руху, щоб програма надіслала `stop`.
11. Перевірте кнопки `LED ON`, `LED OFF` і `Ping`.
12. Використовуйте код `simple_robot_demo.py` як просту основу для власного графічного інтерфейсу.
